## 4. RL-genuineness ranking (most RL training -> least)

All three candidates were trained with the same recipe: --policy-anchor (policy
head distilled to BC seed, only trunk+value head shaped by real game outcomes),
shaping with terminal_weight=0.8 (80% real outcome / 20% potential-based shaping),
batch=4096, sample_per_step=8192, lr=2e-5. So qualitatively identical RL.

Quantitative ranking (gradient steps with real game-outcome reward signal):

For the academic report ("more RL"):
  - new best.pt vs deployed: 4.4x more gradient steps under the same RL recipe.
  - new final.pt vs deployed: 7.4x more gradient steps.
  - But: the eval-gating in train.py promoted step 704 as best; the value-loss
    curve plateaued at ~0.39 around step 100 and never re-dropped (Fig: see
    health.jsonl). So extra training past the eval peak did not measurably
    improve the value head.
